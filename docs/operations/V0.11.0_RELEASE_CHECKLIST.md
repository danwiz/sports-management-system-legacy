## GitHub release body

```markdown
## Sports Management System — v0.11.0

This maintenance release completes service-routed lifecycle operations, full CLI editing workflows, formal requirements traceability, and visual architecture documentation.

### Lifecycle integrity

- Routes member, match, and performance mutations through `SportsService`.
- Adds referentially safe member and match deletion.
- Adds validated performance update and deletion.
- Prevents rejected mutations from changing persisted state.

### Complete editing workflows

- Adds team editing through the validated service layer.
- Expands match editing to sport, teams, schedule, venue, and scores.
- Preserves existing values when optional edit fields are left blank.
- Adds safe optional integer parsing.
- Uses sport-specific performance labels and prompts.

### Engineering evidence

- Adds an authoritative requirements traceability matrix.
- Adds Mermaid diagrams for the domain model, component architecture, mutation sequence, backup/restore flow, and preservation boundary.
- Updates the README and GitHub Pages case study with architecture and traceability links.

### Validation

- CMake configuration: passed
- C++20 build: passed
- All registered CTest suites: passed
- GitHub Actions release validation: passed

### Compatibility

This release retains the CSV data model, CMake build, CPack packaging, backup/restore workflow, governance controls, and preserved legacy artifacts established in `v0.10.0`.
```
