# v0.11.0 Modernization Roadmap

## Release objective

Complete the service-routed lifecycle operations and strengthen traceability, usability, and architecture documentation without modifying preserved legacy artifacts.

## Planned outcomes

### 1. Complete entity lifecycle operations

- Route member deletion through `SportsService`.
- Route match deletion through `SportsService`.
- Add update and removal operations for performance records.
- Enforce referential integrity and consistent validation across all write paths.
- Add regression coverage for successful and rejected operations.

### 2. Complete interactive editing

- Add a team-edit workflow to the CLI.
- Expand match editing beyond score changes to teams, sport, date/time, and venue.
- Replace generic performance-stat prompts with sport-specific labels and guidance.
- Preserve explicit Back and Exit navigation and safe line-based input handling.

### 3. Requirements traceability

- Create a requirements traceability matrix covering the original assignment, UAT feedback, modern implementation, tests, and unresolved backlog.
- Mark archival-only, implemented, partially implemented, and deferred requirements.
- Link acceptance evidence to test names and source locations.

### 4. Architecture documentation

- Add Mermaid class, component, sequence, and data-flow diagrams.
- Document boundaries among CLI, application services, repositories, domain models, and CSV storage.
- Record the decision to keep archival source isolated from the modernization.

### 5. Release quality

- Keep GitHub Actions green.
- Update release notes and changelog.
- Validate installation and CPack outputs.
- Publish `v0.11.0` only after all acceptance criteria pass.

## Acceptance criteria

- All write and delete operations use service-layer validation.
- CLI supports full team and match editing.
- Sport-specific statistic prompts are clear and tested.
- Traceability matrix maps recovered requirements to implementation and evidence.
- Mermaid architecture diagrams render correctly on GitHub.
- Existing seven tests remain green and new regression tests pass.
- `v0.11.0` packaging is reproducible.
