# v0.11.0 Increment 2 — Complete Editing Workflows

## Scope completed

- Added team editing through `SportsService::update_team`.
- Expanded match editing to sport, home team, away team, scheduled date/time, venue, home score, and away score.
- Preserved blank-to-retain behavior for existing values.
- Added safe optional integer input for score and statistic edits.
- Added sport-specific statistic labels:
  - Soccer: goals, assists, saves
  - Netball: goals, assists, interceptions
  - Track and field: first-, second-, and third-place finishes
- Applied sport-specific labels to listing, creation, and editing workflows.
- Extended service regression coverage for successful and rejected team/match updates.

## Integrity behavior

- A team sport cannot be changed while members or matches reference that team.
- A match update is rejected when either selected team does not exist or its sport conflicts with the match sport.
- Rejected updates leave the stored entity unchanged.
- All mutations continue to route through `SportsService`.

## Validation target

The combined Increment 1 and Increment 2 validation must pass CMake configuration, the C++20 build, and all registered CTest suites from a clean GitHub Actions checkout.
